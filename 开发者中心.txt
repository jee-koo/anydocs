简介
Cregis 致力于为企业提供创新的金融工具，帮助企业轻松且安全地管理数字资产。我们的产品套件为企业提供强大的功能，使他们能够轻松管理、交易和利用加密货币。无论您的企业是希望整合加密货币解决方案，还是提升现有的财务运营效率，Cregis都通过最先进的技术助您实现目标。从自托管钱包到支付引擎和金融服务，Cregis将帮助您的企业自信且高效地应对不断发展的数字经济。
WaaS 项目
Cregis钱包基础设施为企业提供即时部署的加密钱包解决方案，支持企业通过WaaS API、桌面客户端和移动应用实现高效可扩展的数字资产管理。无论是基础钱包服务还是复杂的企业级资产管理，Cregis钱包均提供基于MPC的多层安全技术，确保您的数字资产安全。
功能
支持种类丰富的加密数字资产的用户充值、提币和配套的查询与通知等服务，并结合AML对资金进行合规筛查。客户可根据自身业务模式，灵活接入并组合相应服务。

交易所

电商

支付引擎
Cregis 支付引擎是一套完整的加密货币支付综合解决方案，为支付服务商、电商平台、游戏平台、零售商等机构提供一体化的加密货币收单服务。该解决方案涵盖商户接入、订单管理、资金归集、合规扫描（AML）、多渠道结算等全流程功能，帮助您快速构建安全可靠的加密支付系统。
业务流程
Cregis支付引擎整体业务流程如下：

收单
您可以通过调用 创建订单 来完成收单操作。Cregis 会创建指定金额的支付订单，并为此订单分配一个临时收款地址，用户需要在规定期限内完成支付。
资金处理
归集：Cregis 会定期将分散在各个收币地址的资金归集到指定的归集地址中，方便统一管理和使用。
KYT扫描：当订单分配的收款地址接收到资金时，将自动触发KYT扫描，确保资金来源安全，满足不同的合规需求。
异常处理和退款：当遇到多付或少付等异常情况时，您可以向客户发起退款或补款等处理方式。
结算
T+0的高效结算能确保交易快速完成。直接将加密货币转账到指定钱包地址，实现快速提取。
订单结算流程请参考此结算信息。
配置结算币种请参考此结算配置教学。

开发前准备
WaaS项目
开始之前
	•	如果您选择使用 WaaS SDK 而不是手动编写 API 请求，请参阅与您选择的编程语言（Java、Go、Php、C#）相对应的 SDK 指南，将 SDK 集成到您的项目中。
设置账户
	•	按照 团队管理 中的说明创建您的团队，如果团队已经创建，请让您的团队管理员在“管理-成员”邀请您加入团队，并将您的角色设置为开发者。
创建项目
	•	在 Cregis 客户端创建您的WaaS项目并进行相应配置。要了解有关创建WaaS项目的更多信息，请参阅API操作手册。
	•	1.登录 Cregis 客户端
	•	2.在左侧导航菜单中点击 WaaS项目 > 创建，在此之前需要先创建一个钱包
	•	3.输入项目名称（最多 20 个字符），选择想要开启的接口类型
	•	4.点击确认进行身份验证，完成创建
	•	5.为确保请求的安全性，请点击 WaaS项目 > 设置 > IP白名单 添加允许请求Cregis API的IP白名单
	•	6.点击 WaaS项目 > 设置 > 开发者 查询API接入信息
支付引擎
设置账户
	•	按照新手入门中的说明创建您的团队，如果团队已经创建，请让您的团队管理员在“管理-成员”邀请您加入团队，并将您的角色设置为开发者。
创建项目
	•	在 Cregis 客户端创建您的支付引擎并进行相应配置。要了解有关创建支付引擎的更多信息，请参阅 创建支付引擎项目。
	•	登录 Cregis 客户端（生产环境）
	•	在左侧导航菜单中点击 支付引擎 > 创建，在此之前需要先 创建一个钱包
	•	输入项目名称（2-20 个字符），进行配置
	•	点击确认进行身份验证，完成创建
	•	为确保请求的安全性，请点击项目 设置 > IP白名单 添加允许请求Cregis API的IP白名单
	•	点击项目 设置 > 开发者 查询API接入信息
签名规则
签名算法
	•	请求参数中除去sign和空值参数外，其他参数按字典序排序.
字典序定义：在英文字典中，排列单词的顺序是先按照第一个字母以升序排列（即 a、b、c……z 的顺序）；如果第一个字母一样，那么比较第二个、第三个乃至后面的字母。如果比到最后两个单词不一样长（比如，sigh 和 sight），那么把短者排在前。
	•	排序完成后，按照”key1value1key2value2…”进行拼接
	•	将项目的API Key拼接到第 2 步中得到的字符串最前面
	•	将第 3 步中获取的字符串使用散列算法取md5并转为小写即为sign，将sign添加到请求参数中

请求和响应
请求
了解如何对 Cregis 服务的 API 请求进行身份验证。API请求接口参数分为两部分：系统参数 和 业务参数。
系统参数:每个 API 调用都需要系统参数

业务参数：特定于各个 API 调用或端点的参数。除了系统参数外，其他参数是业务参数，以下是一个请求示例。
客户端使用 Postman 示例:

Call URL：
curl --location --request POST 'https://xx.xx.xx/api/v1/payout' \
--header 'Content-Type: application/json' \
--data-raw '{
    "pid": 1394705669275648,
    "currency": "195@195",
    "address": "TXsmKpEuW7qWnXzJLGP9eDLvWPR2GRn1FS",
    "amount": "0.1",
    "remark": "payout",
    "third_party_id": "1828fc39daaf4d318777e1211c055edf",
    "callback_url": "http://xxx.com/payout/callback",
    "nonce": "rR7ngZ",
    "timestamp": 1709708726668,
    "sign": "e95ba1a7e9bcadb92d9c3405a932715d"
}'
响应
code  string
响应状态码，code为00000表示响应成功，其他状态码表示响应失败。 具体系统错误码可以参考 错误代码
msg  string
错误消息
data  object
业务数据，为 Json 对象，如果请求错误，该字段为空。

工作区管理
请在产品配置过程中，确保已遵循以下安全建议，以最大程度保障团队的资产与操作安全。Cregis Wallet 提供多种风控机制，旨在解决存储、管理和转移数字资产中遇到的挑战。
	•	管理用户
	•	工作区允许您邀请团队成员参与工作区管理
	•	设置白名单
	•	API白名单：
	•	地址簿：
	•	设备和IP白名单：
	•	设置策略引擎
	•	职责分离
	•	为了确保工作区运营的安全，合理划分职责和委派责任至关重要。第一步也是最重要的一步是区分处理关键管理任务的用户和管理常规操作的用户。
	•	对于前者，了解操作权限级别以及他们对工作区所做的任何更改将产生的影响至关重要。请参阅此文了解更多信息。
	•	对于日常运营，在分配职责时要考虑交易过程中的三个关键角色：
	•	谁来发起交易？
	•	谁来审批交易？
	•	谁来签署交易？
	•	手动流程：通常用于涉及大量资产的操作，这需要 100% 的人工干预。交易将通过 Cregis 客户端发起，由特定人员在其客户端上手动审批，并由指定人员签名。请注意，发起者和审批者无需拥有 MPC 分片即可审批和发起交易。
	•	自动化流程：通常用于小额提现，由API发起。系统根据设定的规则条件，自动对满足条件的交易进行签名。请注意，如交易触发策略进入审批，需由特定人员在其客户端上手动审批。

WaaS 项目
地址
创建子地址
要接收代币，您需要在对应WaaS项目中创建地址。此操作在指定的WaaS 项目内生成一个地址。
POST /api/v1/address/create
Ton、Xrp不支持请求API创建地址，请在客户端手动创建，详情参考创建子地址
请求参数
pid  int64    必须
您创建的WaaS项目ID，请前往“Cregis 客户端 > WaaS项目 > 选择对应项目 > 设置 > 开发者”查看
chain_id  string    必须
区块链的 ID，请确认钱包中已添加您想要的主链，查询区块链ID请参阅 支持的代币
alias  string    非必须
地址别名（1-40个字符），在Cregis 客户端显示
callback_url  string    非必须
回调地址,当地址接收资金时，WaaS 服务会向对应的callback_url发起交易回调通知,回调详情参考 Wabhook 和回调
nonce  string    必须
6位随机字符串
timestamp  int64    必须
当前请求时间戳，单位毫秒
sign  string    必须
请求的数字签名，防止参数被非法篡改，签名规则参考 签名规则 
请求示例：
{
  "pid": 1382528827416576,
  "callback_url": "http://xxxx.com/deposit/callback",
  "chain_id": "60",
  "alias": "cc",
  "nonce": "ubqso3",
  "timestamp": 1687850657960,
  "sign": "f5be13fdd8c6f63951ca4427359457cb"
}
返回结果
{
  "code": "00000",
  "msg": "ok",
  "data": {
    "address": "TBotPUNNgeSJmqWkSE7JoHdAnsdMyVnUca"
  }
}
